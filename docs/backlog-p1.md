# pin-stitch P1/P2 백로그

본 문서는 `docs/prd-v2.md` 기준 MVP 이후로 미룬 기능을 관리한다.

## 상태 정의

- `BACKLOG`: 미착수
- `READY`: 다음 스프린트 후보
- `IN_PROGRESS`: 진행 중
- `DONE`: 완료

## P1 백로그

| ID | 영역 | 항목 | 설명 | 상태 |
| --- | --- | --- | --- | --- |
| BL-P1-001 | Review | 리뷰 작성 | 구매자 체형 정보와 착용 사이즈를 포함한 리뷰 작성 기능 | BACKLOG |
| BL-P1-002 | Return | 반품 사유 입력 | 주문 상품별 반품 사유를 구조화하여 수집 | BACKLOG |
| BL-P1-003 | Insight | 반품 사유 자동 분류 | 텍스트 반품 사유를 사이즈/핏/소재/색상 등으로 분류 | BACKLOG |
| BL-P1-004 | Recommendation | 체형별 추천 고도화 | 유사 체형 만족도와 핏 점수를 기반으로 추천 영역 제공 | BACKLOG |
| BL-P1-005 | Seller | 상품 개선 가이드 자동 생성 | 리뷰/반품 분석 결과를 상세페이지 개선 문구로 변환 | BACKLOG |
| BL-P1-006 | Seller | 리포트 다운로드 | 셀러 분석 결과를 CSV 또는 PDF로 다운로드 | BACKLOG |
| BL-P1-007 | Commerce | 실제 결제 연동 | 주문 생성 흐름을 실제 결제 플로우로 확장 | BACKLOG |
| BL-P1-008 | Store | 실제 재고 연동 | 샘플 매장 재고를 POS/ERP 기반 재고 연동으로 확장 | BACKLOG |

| BL-P2-005 | AI | 개인 스타일링 챗봇 | 체형/취향/보유 상품 기반 스타일링 상담 | BACKLOG |

## 우선순위 기준

- 고객 구매 판단에 직접 기여하는가
- 셀러가 상품 개선에 활용할 수 있는가
- 고객 또는 셀러에게 제공하는 제품 가치가 명확한가
- 기존 MVP 데이터 구조를 크게 흔들지 않는가

## 다음 스프린트 우선 후보

1. `BL-P1-001` 리뷰 작성
2. `BL-P1-002` 반품 사유 입력
3. `BL-P1-005` 상품 개선 가이드 자동 생성

세 항목은 MVP의 체형/리뷰/셀러 인사이트 흐름을 자연스럽게 강화하므로 우선순위가 높다.
