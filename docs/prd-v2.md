## 2. 제품 개요

`pin-stitch`는 체형 정보와 구매 리뷰를 연결해 사용자가 자신에게 맞는 패션 상품을 더 쉽게 고르도록 돕는 커머스 플랫폼이다.

사용자는 자신의 체형과 비슷한 구매자의 리뷰를 중심으로 상품을 판단하고, 셀러는 리뷰와 반품 데이터를 분석해 상품 상세페이지, 옵션 구성, 상품 기획을 개선할 수 있다.

## 3. 문제 정의

### 고객 문제

- 내 체형에 맞는 상품인지 구매 전 판단하기 어렵다.
- 리뷰가 많아도 나와 비슷한 체형의 후기를 찾는 데 시간이 오래 걸린다.
- 사이즈, 기장, 어깨, 허리, 소재감 관련 정보가 리뷰 곳곳에 흩어져 있다.
- 가까운 매장에 원하는 사이즈가 있는지 확인하기 어렵다.

### 셀러 문제

- 체형별 만족도와 불만 원인을 한눈에 보기 어렵다.
- 리뷰와 반품 사유가 상품 개선에 연결되지 않는다.
- 반복되는 사이즈/핏 문제로 반품 비용과 고객 이탈이 발생한다.

## 4. 목표 / 비목표

### 목표

- 사용자가 본인과 유사한 체형의 리뷰를 빠르게 확인할 수 있게 한다.
- 상품별 체형 적합도와 사이즈 주의사항을 제공한다.
- 샘플 매장 재고를 보여줘 온라인 탐색과 오프라인 피팅을 연결한다.
- 셀러가 상품별 체형 만족도와 핏 불만 원인을 파악할 수 있게 한다.

### 비목표

- MVP 단계에서 실제 결제 연동을 필수로 구현하지 않는다.
- MVP 단계에서 사진 기반 체형 분석을 구현하지 않는다.
- MVP 단계에서 AI Virtual Try-on을 구현하지 않는다.
- MVP 단계에서 실제 POS/ERP 재고 연동을 구현하지 않는다.

## 5. 사용자 유형

| 사용자 | 설명 |
| --- | --- |
| Customer | 패션 상품을 탐색하고 구매하는 일반 사용자 |
| Seller | 입점 상품의 리뷰/반품 데이터를 확인하고 상품을 개선하는 브랜드/셀러 |
| Admin | 샘플 데이터와 운영 상태를 관리하는 플랫폼 운영자 |

## 6. 핵심 시나리오

- UC-001: 사용자가 체형 프로필을 등록하고 상품을 탐색한다.
- UC-002: 사용자가 상품 상세에서 나와 비슷한 체형의 리뷰 요약을 확인한다.
- UC-003: 사용자가 체형 적합도와 사이즈 주의사항을 보고 옵션을 선택한다.
- UC-004: 사용자가 가까운 매장의 색상/사이즈 재고를 확인한다.
- UC-005: 셀러가 상품별 체형 만족도와 부정 키워드를 확인한다.
- UC-006: 셀러가 반품 사유와 리뷰 분석을 기반으로 상품 개선 인사이트를 확인한다.

## 7. 범위 정의

### P0

- 고객 체형 프로필 등록/수정
- 상품 목록/검색/필터/정렬
- 상품 상세
- 리뷰 목록 및 체형별 필터
- 체형 기반 리뷰 요약
- 체형 적합도 점수
- 사이즈/핏 주의사항
- 샘플 매장 재고 안내
- 장바구니/주문 생성
- 장바구니 쿠폰 자동 최적 적용
- 셀러 상품 분석 대시보드
- 셀러 리뷰/반품 인사이트

### P1

- 리뷰 작성
- 반품 사유 입력/분류
- 체형별 추천 상품 영역
- 셀러 리포트 다운로드
- 상품 개선 가이드 자동 생성
- 쿠폰 자동 적용 정책 고도화
- 실제 주문/결제 연동

### P2

- 사진 기반 체형 분석
- AI Virtual Try-on
- 실제 매장 재고 연동
- 스마트 피팅룸 요청
- 개인 스타일링 챗봇

## 9. 비기능 요구사항

### 성능

- `GET /products` p95 900ms 이하
- `GET /products/:productId` p95 900ms 이하
- `GET /products/:productId/review-summary` p95 1200ms 이하
- `GET /seller/products/:productId/insights` p95 1200ms 이하
- 상품 목록/상세 LCP 2.8s 이하

### 접근성

- 키보드 내비게이션 지원
- 명확한 포커스 상태 제공
- 이미지 대체 텍스트 제공
- 텍스트 대비 WCAG AA 수준 지향

### 보안/개인정보

- 체형 정보는 사용자 동의 기반으로 수집한다.
- 셀러 대시보드에서는 개인 식별 가능한 리뷰어 정보를 노출하지 않는다.
- 관리자/셀러 권한을 분리한다.

### 관측성

- 상품 상세 진입, 리뷰 요약 클릭, 체형 필터 사용, 매장 재고 클릭 이벤트를 수집한다.
- 셀러 대시보드 조회와 인사이트 카드 클릭 이벤트를 수집한다.

## 12. 리스크 및 대응

| Risk | 영향도 | 대응 |
| --- | --- | --- |
| 초기 리뷰 데이터 부족 | High | 샘플 데이터로 초기 기능을 구축한 뒤 리뷰 작성 기능 확장 |
| 체형 정보 입력 저항 | Medium | 필수 입력 최소화, 선택 입력 분리 |
| 리뷰 요약 신뢰도 부족 | Medium | 요약 근거 리뷰를 함께 노출 |
| 재고 데이터 현실성 부족 | Medium | MVP에서는 샘플 데이터임을 명확히 하고 실제 연동은 P1로 분리 |
| 개인정보 우려 | High | 체형 정보 선택 동의, 셀러 화면 비식별 처리 |

## 13. 릴리스 계획

- 1주차: 도메인/API/샘플 데이터 설계
- 2주차: 고객 체형 프로필, 상품 목록/상세 구현
- 3주차: 리뷰 필터/요약, 체형 적합도 구현
- 4주차: 매장 재고 안내, 셀러 대시보드 구현
- 5주차: 장바구니/주문 흐름, QA, 성능/접근성 점검

## 14. 오픈 이슈

- Q1: MVP에서 장바구니/주문을 P0로 유지할지, 상품 탐색/분석에 더 집중할지 결정
- Q2: 체형 적합도 점수 산식의 첫 버전을 문서화해야 한다.
- Q3: 리뷰 요약을 규칙 기반으로 시작할지 LLM API 기반으로 시작할지 결정
