# pin-stitch MVP 구현 계획서

## 1. 구현 목표

이번 MVP의 목표는 체형 기반 상품 탐색부터 장바구니, 주문 생성, 셀러 인사이트까지 이어지는 패션 커머스 핵심 흐름을 완성하는 것이다.

핵심 경험은 다음 한 문장으로 정의한다.

> 사용자는 체형 프로필을 등록한 뒤 나와 비슷한 체형의 리뷰와 적합도 점수를 보고 상품을 고르며, 장바구니와 주문 생성까지 완료한다. 셀러는 누적된 리뷰/반품 데이터를 상품 개선 인사이트로 확인한다.

## 2. MVP 범위

### 포함 범위

- 고객 체형 프로필 등록/수정
- 상품 목록/검색/필터/정렬
- 상품 상세
- 체형별 리뷰 필터링
- 체형 기반 리뷰 요약
- 체형 적합도 점수
- 사이즈/핏 주의사항
- 매장별 샘플 재고 안내
- 장바구니
- 쿠폰 자동 최적 적용
- 주문 생성 및 주문 완료
- 주문 내역/주문 상세
- 셀러 대시보드
- 상품별 리뷰/핏/반품 인사이트

### 제외 범위

- 실제 결제 PG 연동
- 실제 배송/운송장 연동
- 실제 POS/ERP 재고 연동
- 사진 기반 체형 분석
- AI Virtual Try-on
- 지오펜스/비콘 기반 매장 모드
- 스마트 피팅룸 요청

## 3. MVP 사용자 흐름

## 3.1 고객 흐름

1. 사용자가 홈에 진입한다.
2. 체형 프로필이 없으면 체형 등록 CTA를 확인한다.
3. 키, 평소 사이즈, 골격 타입, 선호 핏을 등록한다.
4. 상품 목록에서 상품을 탐색한다.
5. 상품 카드에서 체형 적합도 배지를 확인한다.
6. 상품 상세에서 유사 체형 리뷰 요약과 사이즈 주의사항을 확인한다.
7. 리뷰 탭에서 나와 비슷한 체형의 리뷰를 필터링한다.
8. 원하는 색상/사이즈 옵션을 선택한다.
9. 매장 재고 섹션에서 가까운 매장 보유 여부를 확인한다.
10. 장바구니에 상품을 담는다.
11. 장바구니에서 상품/옵션/수량과 자동 적용된 쿠폰 할인을 확인한다.
12. 배송지를 입력하고 주문을 생성한다.
13. 주문 완료 화면과 주문 내역에서 주문 상태와 할인 내역을 확인한다.

## 3.2 셀러 흐름

1. 셀러가 대시보드에 진입한다.
2. 전체 상품의 평균 평점, 핏 불만 비율, 반품률을 확인한다.
3. 문제 상품 목록에서 이슈가 큰 상품을 선택한다.
4. 상품별 체형 만족도와 사이즈별 핏 평가 분포를 확인한다.
5. 긍정/부정 리뷰 키워드를 확인한다.
6. 상품 개선 인사이트 카드를 확인한다.
7. 상세페이지 보완, 사이즈 안내 강화, 옵션 개선 등의 액션을 결정한다.

## 6. 핵심 계산 로직

## 6.1 유사 체형 리뷰 매칭

사용자의 `BodyProfile`과 리뷰어 정보를 비교해 유사 리뷰를 찾는다.

초기 규칙:

- 키 차이 5cm 이내면 높은 가중치
- 같은 골격 타입이면 높은 가중치
- 같은 평소 사이즈 또는 구매 사이즈면 중간 가중치
- 같은 선호 핏이면 중간 가중치

## 6.2 체형 적합도 점수

`FitScore`는 0~100점으로 계산한다.

초기 산식 예시:

```text
FitScore =
  유사 체형 평균 별점 점수 40%
  + 정사이즈 성공 비율 25%
  + 핏 관련 부정 키워드 역점수 20%
  + 반품 사유 역점수 15%
```

신뢰도 기준:

- 유사 체형 리뷰 15개 이상: `HIGH`
- 유사 체형 리뷰 5~14개: `MEDIUM`
- 유사 체형 리뷰 1~4개: `LOW`
- 유사 체형 리뷰 없음: 전체 리뷰 기반 fallback

## 6.3 리뷰 요약

MVP에서는 LLM 연동 없이 규칙 기반 요약으로 시작한다.

요약 기준:

- 유사 체형 리뷰에서 자주 등장한 긍정 키워드
- 유사 체형 리뷰에서 자주 등장한 부정 키워드
- `SMALL`, `TRUE_TO_SIZE`, `LARGE` 비율
- 사이즈/어깨/허리/기장/소재 키워드 빈도

예시:

```text
160cm 전후 WAVE 체형 사용자는 허리 라인은 만족하지만 어깨가 약간 여유롭다는 의견이 많습니다.
```

## 6.4 사이즈/핏 주의사항

부정 키워드와 핏 평가를 기반으로 경고 문구를 만든다.

예시:

- `어깨` 부정 키워드가 많으면: "어깨가 좁은 사용자는 상체가 커 보일 수 있습니다."
- `기장` 부정 키워드가 많으면: "160cm 이하 사용자는 기장이 길게 느껴질 수 있습니다."
- `SMALL` 비율이 높으면: "정사이즈보다 작다는 리뷰가 많습니다."

## 6.5 셀러 인사이트

상품별 리뷰와 반품 데이터를 집계해 인사이트를 만든다.

초기 인사이트 유형:

- `DETAIL_PAGE_IMPROVEMENT`: 상세페이지 정보 보완
- `SIZE_GUIDE_IMPROVEMENT`: 사이즈 가이드 보완
- `OPTION_IMPROVEMENT`: 옵션/사이즈 구성 개선
- `TARGETING_OPPORTUNITY`: 특정 체형 대상 추천 강화

예시:

```text
WAVE 체형 고객의 어깨 여유 관련 불만이 반복됩니다. 어깨 실측과 착용 컷을 상세페이지 상단에 보강해야 합니다.
```

## 6.6 쿠폰 자동 최적 적용

장바구니의 상품 목록과 사용자가 보유한 쿠폰을 기준으로 최종 주문 예정 금액이 가장 낮아지는 쿠폰 조합을 계산한다.

MVP 쿠폰 유형:

- 주문 전체 정액 할인
- 주문 전체 정률 할인
- 특정 상품 할인
- 특정 카테고리 할인
- 특정 브랜드 할인

초기 적용 규칙:

- 사용 기간이 지난 쿠폰은 제외한다.
- 최소 주문 금액을 만족하지 못한 쿠폰은 제외한다.
- 상품/카테고리/브랜드 대상 조건을 만족하지 못한 쿠폰은 제외한다.
- `stackable`이 false인 쿠폰은 다른 쿠폰과 함께 적용하지 않는다.
- 상품 쿠폰은 적용 가능한 상품 라인에만 할인 금액을 배분한다.
- 정률 쿠폰은 `maxDiscountAmount`가 있으면 최대 할인 금액을 넘지 않는다.
- 여러 조합의 최종 금액이 같으면 만료일이 빠른 쿠폰을 우선 적용한다.

완료 기준:

- 장바구니 진입 시 사용 가능한 쿠폰 조합이 자동 계산된다.
- 사용자는 수동 선택 없이 최대 할인 금액과 최종 주문 예정 금액을 확인할 수 있다.
- 조건을 만족하지 못한 쿠폰은 제외 사유와 함께 표시된다.

## 7. 장바구니/주문 MVP 정책

## 7.1 장바구니

- 사용자는 상품 상세에서 선택한 옵션을 장바구니에 담을 수 있다.
- 같은 옵션을 다시 담으면 수량을 증가시킨다.
- 품절 옵션은 장바구니 담기를 차단한다.
- 장바구니에서는 수량 변경과 삭제가 가능하다.
- 상품 금액 합계는 상품 가격과 수량으로 계산한다.
- 보유 쿠폰 중 적용 가능한 쿠폰 조합을 자동으로 계산한다.
- 최종 주문 예정 금액은 상품 금액에서 쿠폰 할인 합계를 차감해 계산한다.
- 배송비, 결제 수수료, 적립금은 MVP에서 제외한다.

## 7.2 주문 생성

- 주문은 실제 결제 없이 생성한다.
- 주문 생성 시 장바구니 스냅샷을 저장한다.
- 주문 생성 시 자동 적용된 쿠폰과 할인 금액을 주문 스냅샷에 저장한다.
- 주문 상태 초기값은 `ORDER_CREATED`로 둔다.
- 주문 생성 후 장바구니는 비운다.
- 주문 완료 화면에서 주문 번호, 상품, 배송지, 총액, 상태를 보여준다.

## 7.3 주문 상태

MVP 주문 상태는 단순하게 시작한다.

- `ORDER_CREATED`: 주문 생성
- `PREPARING`: 상품 준비 중
- `SHIPPED`: 배송 중
- `DELIVERED`: 배송 완료
- `CANCELLED`: 주문 취소

실제 결제 상태, 환불 상태, 배송 추적은 P1 이후로 미룬다.

## 9. 구현 단계

## 9.1 1단계: 데이터와 도메인 로직

목표: 화면 개발 전에 기능의 근간이 되는 데이터를 만든다.

- 도메인 타입 정의
- 샘플 상품 데이터 작성
- 샘플 리뷰 데이터 작성
- 샘플 매장/재고 데이터 작성
- `calculateFitScore` 구현
- `matchSimilarReviews` 구현
- `generateReviewSummary` 구현
- `generateSellerInsights` 구현

완료 기준:

- 하나의 상품 상세에서 체형 적합도와 리뷰 요약을 계산할 수 있다.
- 하나의 셀러 상품 분석 화면에 필요한 집계 데이터를 만들 수 있다.

## 9.2 2단계: 고객 상품 탐색

목표: 사용자가 체형 기반으로 상품을 탐색하고 판단할 수 있게 한다.

- 체형 프로필 화면
- 상품 목록 화면
- 상품 상세 화면
- 리뷰 필터 화면
- 매장 재고 화면

완료 기준:

- 체형 프로필을 등록하면 상품 목록/상세의 적합도와 요약이 달라진다.
- 상품 상세에서 장바구니 담기 전까지의 의사결정 정보가 완성된다.

## 9.3 3단계: 장바구니/주문

목표: 상품 선택 이후 커머스 기본 흐름을 완성한다.

- 장바구니 조회/수량 변경/삭제
- 쿠폰 자동 최적 적용
- 주문 생성 화면
- 주문 완료 화면
- 주문 내역/주문 상세 화면

완료 기준:

- 상품 상세에서 장바구니에 담은 상품에 최대 할인 쿠폰 조합이 자동 적용된다.
- 쿠폰 할인 내역을 포함한 장바구니를 주문으로 생성할 수 있다.
- 주문 생성 후 주문 내역에서 확인할 수 있다.

## 9.4 4단계: 셀러 인사이트

목표: 고객 리뷰/반품 데이터를 셀러 의사결정 화면으로 연결한다.

- 셀러 대시보드
- 셀러 상품 분석
- 셀러 리뷰 분석
- 셀러 반품 분석

완료 기준:

- 셀러가 문제 상품과 주요 핏 불만을 확인할 수 있다.
- 상품별 개선 인사이트가 최소 1개 이상 표시된다.

## 9.5 5단계: QA와 운영 문서 정리

목표: 주요 사용자 흐름과 예외 상태를 안정적으로 점검한다.

- 주요 화면 반응형 점검
- Empty/Loading/Error 상태 점검
- 샘플 사용자 시나리오 작성
- README 사용자 흐름 정리
- 기능별 운영 기준 정리

완료 기준:

- 고객 플로우와 셀러 플로우가 끊김 없이 동작한다.
- 주요 화면의 로딩, 빈 상태, 오류 상태가 정의되어 있다.

## 11. 제품 완성 기준

### 커머스 흐름

- 상품, 옵션, 재고, 장바구니, 주문의 기본 흐름을 구현한다.
- 여러 쿠폰 조건을 검증하고 최대 할인 조합을 자동 계산한다.
- 상품 상세의 구매 의사결정 정보를 설계한다.
- 주문 생성과 주문 상태를 단순하지만 일관된 도메인으로 관리한다.

### 데이터 기반 UX

- 리뷰 데이터를 체형 기준으로 필터링한다.
- 체형 적합도 점수를 계산해 상품 탐색에 반영한다.
- 요약과 주의사항으로 리뷰 탐색 비용을 줄인다.

### 셀러 경험

- 고객 리뷰와 반품 데이터를 셀러 인사이트로 연결한다.
- 상품별 문제 원인을 체형/사이즈/키워드 기준으로 분석한다.
- 고객 웹과 셀러 웹을 함께 설계해 양면 플랫폼 경험을 보여준다.

### 현실적인 AI 활용 가능성

- MVP는 규칙 기반으로 구현해 안정적으로 동작시킨다.
- 이후 LLM 리뷰 요약, 반품 사유 자동 분류, 상품 개선 문구 생성으로 확장할 수 있다.

## 12. 우선 결정해야 할 사항

- Q1: MVP에서 인증을 실제 로그인으로 구현할지, 샘플 사용자 전환으로 처리할지 결정
- Q2: 주문 데이터를 로컬 상태로 둘지, API 서버와 DB까지 구현할지 결정
- Q3: 리뷰 요약을 규칙 기반으로만 구현할지, LLM API 연동 옵션을 둘지 결정
- Q4: 셀러 웹을 별도 앱으로 분리할지, 고객 앱 내부 `/seller` 라우트로 시작할지 결정
- Q5: 장바구니/주문을 P0 핵심으로 둘 경우, 리뷰 작성 기능은 P1로 유지할지 결정

## 13. 구현 전략

초기에는 실제 백엔드보다 **도메인 로직 + Mock API + 화면 완성도**에 집중한다.

권장 순서는 다음과 같다.

1. 샘플 데이터와 계산 로직을 먼저 만든다.
2. 고객 상품 탐색 화면을 완성한다.
3. 장바구니와 주문 생성으로 커머스 흐름을 닫는다.
4. 같은 데이터를 셀러 대시보드에서 다시 해석한다.
5. 마지막에 실제 API/DB 구조로 옮길 수 있도록 타입과 경계를 정리한다.

이 순서는 고객 구매 판단 흐름을 먼저 완성한 뒤, 같은 데이터를 셀러 인사이트로 확장하기 위한 구현 순서다.
