# pin-stitch UI 가이드

## 1. 디자인 원칙

### 고객 웹

- 상품 이미지가 첫 번째 정보가 되도록 구성합니다.
- 체형 적합도, 리뷰 요약, 사이즈 주의사항은 상품 상세의 핵심 판단 정보로 노출합니다.
- 장바구니와 주문 화면은 할인, 총액, 옵션 정보를 명확히 보여줍니다.
- 과도한 장식보다 상품, 가격, 리뷰, 재고 정보의 가독성을 우선합니다.
- 모바일에서도 옵션 선택, 리뷰 확인, 장바구니 이동이 끊기지 않아야 합니다.

### 셀러 웹

- 고객 웹보다 정보 밀도를 높게 가져갑니다.
- 지표, 표, 차트, 인사이트 카드를 빠르게 스캔할 수 있어야 합니다.
- 감성적인 비주얼보다 데이터 비교와 문제 식별을 우선합니다.
- 셀러가 바로 실행할 수 있는 개선 제안을 명확히 구분합니다.

- 화면 섹션 자체를 카드처럼 겹겹이 감싸지 않습니다.

- 원화는 천 단위 구분 기호를 사용합니다.
- 할인 전 금액은 보조 텍스트와 취소선으로 표시합니다.

## 6. 화면별 요구사항

### 데이터 표시 원칙

사용자 화면에는 내부 추적용 데이터, 운영용 데이터, 개발 편의 데이터를 그대로 노출하지 않습니다. 화면에는 사용자가 의사결정하거나 문의할 때 필요한 정보만 사람이 이해하기 쉬운 표현으로 표시합니다.

| 내부 데이터 | 사용자 화면 표시 |
| --- | --- |
| `reviewId`, `basisReviewIds` | 리뷰 수, 대표 리뷰, 체형 조건 |
| `userId` | 비식별 리뷰어, `비슷한 체형의 구매자` 같은 문구 |
| `bodyProfileId` | 노출하지 않음 |
| `productId`, `variantId` | 상품명, 색상, 사이즈, 필요한 경우 SKU 일부 |
| `cartId`, `cartItemId` | 노출하지 않음 |
| `couponId` | 쿠폰명, 할인 금액, 적용 조건 |
| `orderId` | 사용자용 주문번호 |
| `orderLineId` | 상품명, 옵션, 수량 |
| `returnReasonId` | `사이즈가 작음`, `기장이 김` 같은 사용자 친화 문구 |
| `insightId` | 인사이트 제목, 내용, 심각도 |
| `requestId` | 일반 화면에는 숨김. 고객센터 문의 또는 오류 상세에서만 제한적으로 표시 |
| 원본 ISO timestamp | `2026.05.19`, `3분 전`, `오늘 업데이트됨` 같은 가공된 시간 |
| `lat`, `lng` | 거리, 지역명, 매장명 |
| 정확한 재고 수량 | 정책에 따라 `재고 있음`, `소량 남음`, `품절` 또는 필요한 경우 수량 |

표시 원칙:

- 내부 ID는 일반 사용자 화면에 직접 노출하지 않습니다.
- 사용자가 문의나 확인에 써야 하는 식별자는 내부 ID와 별도로 사용자용 번호를 제공합니다.
- 시간, 위치, 상태, 수량은 사람이 이해할 수 있는 표현으로 가공합니다.
- 운영/디버깅 정보는 관리자 화면, 로그, 고객센터용 오류 상세에만 둡니다.
- 셀러 화면에서도 고객의 개인 식별 정보는 노출하지 않고 체형 그룹, 사이즈, 핏 평가처럼 비식별 정보로 표시합니다.

### 홈 `/`

- 추천 상품 섹션
- 인기 카테고리/브랜드 진입 링크
- 체형 프로필 미등록 사용자의 등록 CTA
- 추천 데이터 조회 실패 시 기본 인기 상품 표시

### 체형 프로필 `/profile/body`

- 키, 몸무게, 평소 상의/하의 사이즈 입력
- 골격 타입: `STRAIGHT`, `WAVE`, `NATURAL`, `UNKNOWN`
- 선호 핏: `SLIM`, `REGULAR`, `LOOSE`
- 체형 정보 활용 동의
- 필수값 누락 또는 동의 미체크 시 저장 차단

### 상품 목록 `/products`

- 검색어 입력
- 카테고리/브랜드/가격 필터
- 체형 적합도 높은 순, 인기순, 최신순, 가격순 정렬
- 상품 카드에 이미지, 브랜드, 상품명, 가격, 체형 적합도 배지, 유사 체형 리뷰 수 표시
- 검색 결과 없음 Empty State
- 체형 프로필 미등록 시 적합도 대신 프로필 등록 CTA

### 상품 상세 `/products/:productId`

- 상품 이미지, 브랜드, 상품명, 가격, 소재, 사이즈표
- 색상/사이즈 옵션 선택
- 체형 적합도 점수와 근거 문장
- 체형 기반 리뷰 요약과 근거 리뷰 수
- 리뷰 요약 근거로 체형 조건과 대표 근거 리뷰를 표시하되, 내부 리뷰 ID는 노출하지 않음
- 사이즈/핏 주의사항
- 리뷰 필터 진입
- 매장 재고 섹션
- 장바구니 담기
- 선택 옵션 품절 시 장바구니 버튼 비활성화

### 리뷰 요약 근거 표시 원칙

- 내부적으로는 `basisReviewIds`로 요약에 사용된 리뷰를 추적합니다.
- 사용자 화면에는 리뷰 ID를 노출하지 않습니다.
- 사용자에게는 요약에 사용된 리뷰 수, 체형 조건, 대표 근거 리뷰를 보여줍니다.
- 대표 근거 리뷰는 리뷰 카드 또는 접을 수 있는 목록 형태로 표시합니다.
- 리뷰어 정보는 키 범위, 골격 타입, 구매 사이즈, 핏 평가처럼 비식별 정보만 표시합니다.
- 예시 문구는 `비슷한 체형 리뷰 18개를 바탕으로 요약했어요.`처럼 사용자 친화적으로 작성합니다.

### 리뷰 탐색 `/products/:productId/reviews`

- 전체 리뷰 목록
- 키 범위, 착용 사이즈, 골격 타입, 선호 핏, 핏 평가 필터
- 리뷰 카드에 별점, 체형 정보, 착용 사이즈, 핏 평가, 텍스트 표시
- 필터 결과 없음 Empty State

### 매장 재고 `/products/:productId/stores`

- 선택한 색상/사이즈 기준 매장별 재고 목록
- 매장명, 거리, 영업 상태, 보유 수량, 층/존/랙 위치
- 재고 없음 시 온라인 배송 옵션 안내
- 위치 권한이 없으면 기본 지역 기준 목록 표시

### 장바구니 `/cart`

- 상품/옵션/수량 표시
- 수량 변경/삭제
- 총 상품 금액
- 자동 적용 쿠폰 목록
- 상품별 할인 금액과 주문 전체 할인 금액
- 적용 제외 쿠폰과 제외 사유
- 최종 주문 예정 금액
- 주문하기 버튼
- 품절 옵션은 주문 진행 차단

### 주문 생성 `/checkout`

- 배송지 입력
- 주문 상품 확인
- 자동 적용된 쿠폰과 할인 금액 확인
- 주문 생성 CTA
- 배송지 필수값 누락 시 주문 생성 차단
- 장바구니의 할인 결과와 주문 생성 화면의 할인 결과 일치

### 주문 완료/상세 `/orders/:orderId`

- 주문 번호
- 주문 상품
- 배송지
- 할인 내역
- 최종 금액
- 주문 상태

### 주문 내역 `/me/orders`

- 주문 목록
- 주문 상태
- 상세 진입
- 주문 내역 없음 Empty State

### 셀러 대시보드 `/seller`

- 전체 상품 수, 평균 평점, 핏 불만 비율, 반품률 지표 카드
- 문제 상품 Top 5
- 최근 인사이트 카드
- 지표 카드는 4개 이하로 시작

### 셀러 상품 분석 `/seller/products/:productId`

- 상품 기본 정보
- 체형별 평균 평점
- 사이즈별 핏 평가 분포
- 긍정/부정 키워드
- 대표 리뷰
- 개선 인사이트 카드

### 셀러 리뷰 분석 `/seller/reviews`

- 상품, 체형, 사이즈, 평점, 키워드 필터
- 리뷰 목록
- 긍정/부정 키워드 요약

### 셀러 반품 분석 `/seller/returns`

- 반품 사유별 비율
- 상품별 반품률
- 체형 그룹별 반복 이슈
- 반품 사유는 코드가 아니라 사용자 친화적인 문구로 표시

## 7. 상태 화면

### Loading

- 카드형 skeleton을 사용합니다.
- 상품 이미지 영역은 고정 비율 skeleton을 사용합니다.
- 대시보드 지표는 카드 skeleton을 사용합니다.

### Empty

원인과 다음 행동을 함께 제공합니다.

- 검색 결과 없음: 필터 초기화 버튼
- 유사 체형 리뷰 없음: 전체 리뷰 보기 버튼
- 적용 가능한 쿠폰 없음: 상품 금액 조건 안내
- 주문 내역 없음: 상품 보러가기 버튼

### Error

- 데이터 조회 실패
- 장바구니 업데이트 실패
- 쿠폰 계산 실패
- 주문 생성 실패

오류 상태에는 재시도 가능 여부를 명확히 표시합니다.

## 8. 접근성 체크리스트

- 모든 버튼은 키보드로 접근 가능해야 합니다.
- 포커스 상태는 시각적으로 표시합니다.
- 상품 이미지는 대체 텍스트를 제공합니다.
- 색상만으로 상태를 구분하지 않습니다.
- 폼 오류는 필드와 가까운 위치에 표시합니다.
- 차트는 핵심 수치를 텍스트로도 제공합니다.
- 모바일에서 CTA가 화면 밖으로 밀리지 않아야 합니다.
- 버튼과 배지의 텍스트가 부모 영역을 넘지 않아야 합니다.

## 9. 구현 검수 체크리스트

- [ ] 상품 카드 이미지 비율이 일정한가
- [ ] 체형 적합도 점수와 근거 문장이 함께 표시되는가
- [ ] 리뷰 요약에 근거 리뷰 수가 표시되는가
- [ ] 유사 체형 리뷰가 없을 때 전체 리뷰 기반 fallback이 동작하는가
- [ ] 쿠폰 자동 적용 결과와 제외 사유가 표시되는가
- [ ] 주문 화면의 최종 금액이 장바구니와 일치하는가
- [ ] 셀러 대시보드의 지표/차트/인사이트가 과밀하지 않은가
- [ ] Empty/Loading/Error 상태가 모든 주요 화면에 있는가
- [ ] 색상만으로 상태를 구분하지 않는가
