# pin-stitch 디자인 시스템 명세서

## 1. 문서 목적

이 문서는 `pin-stitch` MVP의 UI를 일관되게 구현하기 위한 디자인 기준을 정의한다.

대상 화면은 고객 웹과 셀러 웹이다.

- 고객 웹: 상품 탐색, 리뷰 확인, 장바구니, 주문 흐름
- 셀러 웹: 상품 분석, 리뷰 분석, 반품 분석, 인사이트 확인

## 2. 디자인 원칙

## 2.1 고객 웹

고객 웹은 패션 상품의 이미지와 구매 판단 정보를 중심으로 구성한다.

원칙:

- 상품 이미지가 첫 번째 정보가 되도록 한다.
- 체형 적합도, 리뷰 요약, 사이즈 주의사항은 상품 상세의 핵심 판단 정보로 노출한다.
- 장바구니와 주문 화면은 할인, 총액, 옵션 정보를 명확히 보여준다.
- 과도한 장식보다 상품, 가격, 리뷰, 재고 정보의 가독성을 우선한다.
- 모바일에서도 옵션 선택, 리뷰 확인, 장바구니 이동이 끊기지 않아야 한다.

## 2.2 셀러 웹

셀러 웹은 운영과 분석을 위한 화면이다.

원칙:

- 정보 밀도를 고객 웹보다 높게 가져간다.
- 지표, 표, 차트, 인사이트 카드를 빠르게 스캔할 수 있어야 한다.
- 감성적인 비주얼보다 데이터 비교와 문제 식별을 우선한다.
- 셀러가 바로 실행할 수 있는 개선 제안을 명확히 구분한다.

## 3. 색상 토큰

## 3.1 기본 색상

| 토큰 | 값 | 용도 |
| --- | --- | --- |
| `color.bg` | `#FFFFFF` | 기본 배경 |
| `color.bgSubtle` | `#F7F7F5` | 보조 배경 |
| `color.surface` | `#FFFFFF` | 카드/패널 배경 |
| `color.surfaceMuted` | `#F1F2EE` | 비활성/요약 영역 |
| `color.text` | `#171717` | 기본 텍스트 |
| `color.textMuted` | `#6B6B66` | 보조 텍스트 |
| `color.textSubtle` | `#9A9A92` | 힌트/부가 정보 |
| `color.border` | `#E2E2DC` | 기본 보더 |
| `color.borderStrong` | `#C9C9C0` | 강조 보더 |

## 3.2 브랜드/상태 색상

| 토큰 | 값 | 용도 |
| --- | --- | --- |
| `color.primary` | `#1E3A34` | 주요 CTA, 선택 상태 |
| `color.primaryHover` | `#162B27` | 주요 CTA hover |
| `color.accent` | `#B86B4B` | 할인, 쿠폰, 강조 정보 |
| `color.info` | `#3B6EA8` | 정보 메시지 |
| `color.success` | `#2F7D55` | 성공/적합 |
| `color.warning` | `#B7791F` | 주의/핏 경고 |
| `color.danger` | `#B42318` | 오류/부정 지표 |

## 3.3 색상 사용 규칙

- 기본 화면은 흰색과 중립 회색을 중심으로 구성한다.
- `primary`는 주요 행동 버튼과 선택 상태에만 사용한다.
- `accent`는 쿠폰, 할인 금액, 자동 적용 상태에 사용한다.
- `success`, `warning`, `danger`는 상태 표시와 분석 지표에 사용한다.
- 한 화면에서 강조 색상은 2개 이상 과하게 사용하지 않는다.

## 4. 타이포그래피

## 4.1 폰트

기본 폰트는 시스템 폰트를 사용한다.

```css
font-family:
  system-ui,
  -apple-system,
  BlinkMacSystemFont,
  "Segoe UI",
  sans-serif;
```

## 4.2 크기 기준

| 토큰 | 크기 | 용도 |
| --- | --- | --- |
| `text-xs` | `12px` | 보조 라벨, 메타 정보 |
| `text-sm` | `14px` | 일반 본문, 폼 라벨 |
| `text-base` | `16px` | 주요 본문 |
| `text-lg` | `18px` | 섹션 제목 |
| `text-xl` | `20px` | 화면 내부 주요 제목 |
| `text-2xl` | `24px` | 페이지 제목 |
| `text-3xl` | `30px` | 고객 홈/상세 상단 제목 |

## 4.3 굵기 기준

| 토큰 | 값 | 용도 |
| --- | --- | --- |
| `font-normal` | `400` | 기본 텍스트 |
| `font-medium` | `500` | 라벨, 버튼 |
| `font-semibold` | `600` | 섹션 제목, 카드 제목 |
| `font-bold` | `700` | 페이지 제목, 핵심 수치 |

## 4.4 규칙

- 버튼, 배지, 테이블 셀의 텍스트는 한 줄에서 잘리지 않도록 최소 너비를 잡는다.
- 대시보드 카드 내부 제목은 `text-sm` 또는 `text-base`를 기본으로 한다.
- 상품 상세 제목은 `text-2xl` 이상을 사용할 수 있다.
- 글자 간격은 기본값을 사용한다.

## 5.4 레이아웃 규칙

- 모바일은 단일 컬럼을 기본으로 한다.
- 데스크톱 상품 상세는 이미지 영역과 구매 정보 영역을 2컬럼으로 구성한다.
- 장바구니/주문 화면은 데스크톱에서 상품 목록과 주문 요약을 2컬럼으로 구성한다.
- 셀러 화면은 사이드 내비게이션 또는 상단 탭 중 하나로 일관되게 구성한다.
- 화면 섹션 자체를 카드처럼 겹겹이 감싸지 않는다.

## 6. 형태와 보더

| 요소 | Radius |
| --- | --- |
| 버튼 | `6px` |
| 입력 필드 | `6px` |
| 상품 카드 | `8px` |
| 대시보드 카드 | `8px` |
| 모달/패널 | `8px` |
| 배지 | `999px` |

규칙:

- 카드 안에 또 다른 카드를 넣지 않는다.
- 반복 목록 아이템은 카드 또는 구분선 중 하나를 선택한다.
- 보더는 `color.border`를 기본으로 사용한다.
- 그림자는 최소화하고, 필요한 경우 아주 약한 그림자만 사용한다.

## 7. 공통 컴포넌트

## 7.1 Button

종류:

- Primary
- Secondary
- Tertiary
- Danger
- Icon Button

상태:

- Default
- Hover
- Disabled
- Loading

규칙:

- 주요 CTA는 한 화면의 주요 흐름당 하나만 둔다.
- 아이콘으로 의미가 명확한 경우 아이콘 버튼을 사용한다.
- 로딩 중에는 중복 제출을 막는다.

## 7.2 Input / Select / Checkbox

규칙:

- 라벨은 항상 표시한다.
- 오류 메시지는 입력 필드 아래에 표시한다.
- 필수 입력은 라벨 옆에 표시한다.
- Select는 옵션 수가 많지 않은 필터와 폼에 사용한다.

## 7.3 Badge

사용 대상:

- 체형 적합도
- 재고 상태
- 주문 상태
- 쿠폰 자동 적용
- 셀러 인사이트 심각도

상태 예시:

- `적합도 높음`
- `재고 있음`
- `자동 적용`
- `주의 필요`
- `High`

## 7.4 Price

규칙:

- 원화는 천 단위 구분 기호를 사용한다.
- 할인 전 금액은 보조 텍스트와 취소선으로 표시한다.
- 할인 금액은 `color.accent`를 사용한다.
- 최종 금액은 굵게 표시한다.

## 8. 고객 웹 패턴

## 8.1 ProductCard

구성:

- 상품 이미지
- 브랜드명
- 상품명
- 가격
- 체형 적합도 배지
- 유사 체형 리뷰 수

규칙:

- 이미지는 고정 비율을 사용한다.
- 상품명은 최대 2줄까지 표시한다.
- 체형 프로필이 없으면 적합도 배지 대신 프로필 등록 CTA를 표시한다.

## 8.2 Product Detail

구성:

- 상품 이미지 갤러리
- 상품명/브랜드/가격
- 색상/사이즈 선택
- 체형 적합도 패널
- 리뷰 요약 패널
- 사이즈/핏 주의사항
- 매장 재고 섹션
- 장바구니 CTA

규칙:

- 데스크톱에서는 옵션 선택과 CTA 영역을 우측에 고정 배치할 수 있다.
- 모바일에서는 장바구니 CTA를 하단 고정 버튼으로 제공한다.
- 리뷰 요약은 근거 리뷰 수를 함께 표시한다.

## 8.3 FitScoreBadge

점수 구간:

| 점수 | 표시 |
| --- | --- |
| `80-100` | 적합도 높음 |
| `60-79` | 적합도 보통 |
| `0-59` | 주의 필요 |

규칙:

- 점수만 단독으로 표시하지 않고 짧은 근거 문장을 함께 제공한다.
- 신뢰도가 낮으면 `리뷰 적음` 보조 라벨을 표시한다.

## 8.4 ReviewSummaryPanel

구성:

- 요약 문장
- 주요 긍정 키워드
- 주요 주의 키워드
- 근거 리뷰 수

규칙:

- 요약 문장은 2~3줄을 넘기지 않는다.
- 주의사항은 경고색을 과하게 쓰지 않고 보더/아이콘으로 구분한다.

## 8.5 StoreInventoryList

구성:

- 매장명
- 거리
- 영업 상태
- 보유 수량
- 층/존/랙 위치

규칙:

- 재고 수량이 0이면 온라인 주문 안내를 표시한다.
- 위치 권한이 없으면 기본 지역 기준 목록을 표시한다.

## 8.6 Cart

구성:

- 장바구니 상품 목록
- 수량 변경
- 상품 금액 합계
- 자동 적용 쿠폰 목록
- 적용 제외 쿠폰 목록
- 최종 주문 예정 금액
- 주문 CTA

규칙:

- 자동 적용된 쿠폰은 `자동 적용` 배지를 표시한다.
- 제외 쿠폰은 접을 수 있는 영역에 표시한다.
- 최종 주문 예정 금액은 주문 CTA 근처에 배치한다.

## 8.7 Checkout

구성:

- 배송지 입력
- 주문 상품 요약
- 쿠폰 할인 요약
- 최종 주문 예정 금액
- 주문 생성 CTA

규칙:

- 장바구니의 할인 결과와 주문 생성 화면의 할인 결과가 일치해야 한다.
- 필수 배송지 정보가 없으면 주문 생성 CTA를 비활성화한다.

## 9. 셀러 웹 패턴

## 9.1 Seller Dashboard

구성:

- 핵심 지표 카드
- 문제 상품 목록
- 최근 인사이트
- 반품/핏 불만 추이

규칙:

- 지표 카드는 4개 이하로 시작한다.
- 문제 상품 목록은 이슈 심각도와 원인을 함께 표시한다.
- 셀러가 다음 액션을 판단할 수 있는 문장을 제공한다.

## 9.2 Seller Product Insight

구성:

- 상품 기본 정보
- 체형별 평균 평점
- 사이즈별 핏 평가 분포
- 긍정/부정 키워드
- 대표 리뷰
- 개선 인사이트

규칙:

- 차트와 표를 같은 섹션에 과도하게 섞지 않는다.
- 개선 인사이트는 별도 카드로 분리한다.
- 심각도는 색상과 텍스트를 함께 사용한다.

## 9.3 Seller Review Analysis

구성:

- 필터 영역
- 리뷰 목록
- 키워드 요약

규칙:

- 필터는 상단 고정 영역에 배치한다.
- 리뷰 목록은 체형, 사이즈, 핏 평가를 한눈에 볼 수 있어야 한다.

## 9.4 Seller Return Analysis

구성:

- 반품 사유별 비율
- 상품별 반품률
- 체형별 반복 이슈

규칙:

- 반품률이 높은 상품은 `warning` 또는 `danger` 상태를 사용한다.
- 반품 사유는 코드가 아니라 사용자 친화적인 문구로 표시한다.

## 10. 상태 화면

## 10.1 Loading

- 카드형 skeleton을 사용한다.
- 상품 이미지 영역은 고정 비율 skeleton을 사용한다.
- 대시보드 지표는 카드 skeleton을 사용한다.

## 10.2 Empty

Empty State는 원인과 다음 행동을 함께 제공한다.

예시:

- 검색 결과 없음: 필터 초기화 버튼
- 유사 체형 리뷰 없음: 전체 리뷰 보기 버튼
- 적용 가능한 쿠폰 없음: 상품 금액 조건 안내
- 주문 내역 없음: 상품 보러가기 버튼

## 10.3 Error

오류 상태는 재시도 가능 여부를 명확히 표시한다.

예시:

- 데이터 조회 실패
- 장바구니 업데이트 실패
- 쿠폰 계산 실패
- 주문 생성 실패

## 11. 접근성 기준

- 모든 버튼은 키보드로 접근 가능해야 한다.
- 포커스 상태는 시각적으로 표시한다.
- 상품 이미지는 대체 텍스트를 제공한다.
- 색상만으로 상태를 구분하지 않는다.
- 폼 오류는 필드와 가까운 위치에 표시한다.
- 차트는 핵심 수치를 텍스트로도 제공한다.

## 12. 구현 체크리스트

- [ ] 상품 카드 이미지 비율이 일정한가
- [ ] 모바일에서 CTA가 화면 밖으로 밀리지 않는가
- [ ] 체형 적합도 점수와 근거 문장이 함께 표시되는가
- [ ] 리뷰 요약에 근거 리뷰 수가 표시되는가
- [ ] 쿠폰 자동 적용 결과와 제외 사유가 표시되는가
- [ ] 주문 화면의 최종 금액이 장바구니와 일치하는가
- [ ] 셀러 대시보드의 지표/차트/인사이트가 과밀하지 않은가
- [ ] Empty/Loading/Error 상태가 모든 주요 화면에 있는가
- [ ] 색상만으로 상태를 구분하지 않는가
