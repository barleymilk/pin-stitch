# pin-stitch 화면별 요구사항 명세 (v2)

## 1. 문서 개요

- 목적: `docs/plan.md` 기준으로 고객 웹/관리자 웹의 화면 요구사항을 정의한다.
- 범위: MVP + P1 확장 범위
- 우선순위 규칙:
  - `P0`: 런칭 필수
  - `P1`: 런칭 후 초기 고도화
- 용어 규칙: 엔티티명은 `docs/plan.md`의 표준 엔티티를 그대로 사용한다.

## 2. 고객 웹

## 2.1 홈 (`/`)

- 목표: 카테고리 탐색 진입과 프로모션 노출
- P0 요구사항
  - 카테고리/브랜드 진입 링크 노출
  - 추천/신상품 섹션 노출
- P1 요구사항
  - 개인화 추천 영역 노출
- 사용 엔티티
  - `Product`, `BrandPartner`
- 예외 처리
  - 홈 데이터 조회 실패 시 재시도 UI 노출

## 2.2 상품 목록 (`/products`)

- 목표: 검색/필터 기반 상품 탐색
- P0 요구사항
  - 카테고리/브랜드/가격 필터
  - 정렬(인기순, 최신순, 가격 오름/내림차순)
  - 페이지네이션 또는 무한 스크롤
- P1 요구사항
  - 재고 있음 필터
- 사용 엔티티
  - `Product`, `Variant`, `Inventory`
- 예외 처리
  - 결과 없음 Empty State 제공

## 2.3 상품 상세 (`/products/:productId`)

- 목표: 구매 의사결정 정보 제공
- P0 요구사항
  - 이미지/설명/사이즈 선택
  - 배송 국가 기준 `PriceQuote` 표시 (상품가 + 세금 + 관세 + 배송비)
  - 장바구니 담기
- P1 요구사항
  - 브랜드/배송/반품 정책 상세 노출
  - 가격 산출 근거(`explain`) 요약 노출
- 사용 엔티티
  - `Product`, `Variant`, `Inventory`, `PriceQuote`, `TaxRule`, `DutyRule`, `ShippingRate`
- 예외 처리
  - 재고 부족 시 구매 비활성화
  - 견적 실패 시 재시도 및 기본 가격 fallback

## 2.4 장바구니 (`/cart`)

- 목표: 결제 직전 주문 구성 확정
- P0 요구사항
  - 수량 변경/삭제
  - 배송 국가 변경 시 `PriceQuote` 재계산
  - 주문 예상 총액 표시
- P1 요구사항
  - 품절 SKU 자동 분리 안내
- 사용 엔티티
  - `Cart`, `Product`, `Variant`, `PriceQuote`
- 예외 처리
  - 가격 재계산 실패 시 마지막 성공 견적 유지 + 경고 메시지

## 2.5 체크아웃 (`/checkout`)

- 목표: 결제 완료
- P0 요구사항
  - 배송지 입력/검증
  - 결제 수단 선택 및 결제 요청
  - 최종 결제 금액 확인
- P1 요구사항
  - 쿠폰/프로모션 적용
- 사용 엔티티
  - `Order`, `OrderLine`, `Payment`, `PriceQuote`
- 예외 처리
  - 결제 실패 시 오류 코드별 메시지 및 재시도 UX

## 2.6 주문 완료/조회 (`/orders/:orderId`, `/my/orders`)

- 목표: 주문 상태 가시성 확보
- P0 요구사항
  - 주문 상세/상태 표시
  - 배송 추적 링크
- P1 요구사항
  - 취소/반품 요청 진입
- 사용 엔티티
  - `Order`, `Shipment`, `Payment`

## 3. 관리자 웹

## 3.1 대시보드 (`/admin`)

- 목표: 운영 핵심 지표 모니터링
- P0 요구사항
  - 매출/주문/취소/정산 지표 카드
- P1 요구사항
  - 이상 징후 알림(마진 급락, 실패 주문 급증)
- 사용 엔티티
  - `Order`, `Settlement`, `WorkflowTask`

## 3.2 상품 관리 (`/admin/products`)

- 목표: 상품 등록/운영 자동화
- P0 요구사항
  - `Product`/`Variant` CRUD
  - 필수값 검증 및 저장 차단
- P1 요구사항
  - `AI Commerce Agent` 기반 등록 보조
- 사용 엔티티
  - `Product`, `Variant`, `Inventory`, `WorkflowTask`

## 3.3 가격 정책 (`/admin/pricing-policies`)

- 목표: 가격 정책 운영
- P0 요구사항
  - `PricePolicy` 생성/수정
  - 국가별 가격 시뮬레이션
- P1 요구사항
  - 정책 변경 이력 조회
- 사용 엔티티
  - `PricePolicy`, `TaxRule`, `DutyRule`, `FxRate`, `ShippingRate`, `TradeAgreementRule`, `PriceQuote`

## 3.4 주문 관리 (`/admin/orders`)

- 목표: 주문 처리 운영
- P0 요구사항
  - 주문 검색/필터
  - 상태 변경/출고 처리
- P1 요구사항
  - 예외 주문 자동 태스크 생성
- 사용 엔티티
  - `Order`, `Shipment`, `WorkflowTask`

## 3.5 정산 관리 (`/admin/settlements`)

- 목표: 파트너 정산 운영
- P0 요구사항
  - 정산 주기별 집계 조회
  - 주문/환불/수수료 상세 Breakdown
- P1 요구사항
  - 정산 확정/보류 워크플로우
- 사용 엔티티
  - `Settlement`, `Payout`, `Invoice`, `Order`

## 4. 공통 UI/기술 요구사항

- 인증/인가
  - 고객/관리자 세션 분리
  - 관리자 Role 기반 접근 제어
- 성능
  - 상품 목록/상세의 초기 로딩 최적화
  - 가격 견적 API 재호출 최소화(디바운스/캐시)
- 접근성
  - 키보드 접근성, 대체 텍스트, 포커스 가시성
- 로깅/분석
  - 화면 진입, CTA 클릭, 결제 실패, 견적 실패 이벤트 트래킹

## 5. 수용 기준 샘플

- 상품 상세 화면
  - 선택한 배송 국가 기준으로 1초 내 `PriceQuote.finalPrice`가 표시된다.
  - 재고 없는 `Variant`는 선택 불가 처리된다.
- 체크아웃 화면
  - 결제 실패 시 `error.code` 매핑 메시지를 노출한다.
  - 결제 성공 시 주문 완료 화면으로 이동한다.

## 6. 우선순위 재정렬 요약

- 고객 웹 P0: 홈, 상품 목록/상세, 장바구니, 체크아웃, 주문 조회
- 관리자 웹 P0: 대시보드, 상품 관리, 가격 정책, 주문 관리, 정산 관리
- P1: 개인화 추천, 가격 설명 UI, 예외 태스크 자동화, 정책 이력, 정산 승인 워크플로우
