# pin-stitch 화면별 요구사항 명세 (v2)

## 1. 문서 개요

- 목적: 체형별 리뷰 분석 기반 패션 커머스의 고객 웹/셀러 웹 화면 요구사항을 정의한다.
- 범위: MVP P0 + 초기 고도화 P1
- 기준 문서: `docs/plan.md`, `docs/prd-v2.md`

## 2. 고객 웹

## 2.1 홈 (`/`)

- 목표: 상품 탐색과 체형 프로필 등록을 자연스럽게 유도한다.
- P0 요구사항
  - 추천 상품 섹션 노출
  - 인기 카테고리/브랜드 진입 링크
  - 체형 프로필 미등록 사용자의 등록 CTA 노출
- P1 요구사항
  - 체형 기반 추천 상품 섹션
- 사용 엔티티
  - `Product`, `BodyProfile`, `FitScore`
- 예외 처리
  - 추천 데이터 조회 실패 시 기본 인기 상품 표시

## 2.2 체형 프로필 (`/profile/body`)

- 목표: 리뷰 필터링과 체형 적합도 계산에 필요한 최소 정보를 수집한다.
- P0 요구사항
  - 키, 몸무게, 평소 상의/하의 사이즈 입력
  - 골격 타입 선택: `STRAIGHT`, `WAVE`, `NATURAL`, `UNKNOWN`
  - 선호 핏 선택: `SLIM`, `REGULAR`, `LOOSE`
  - 개인정보/체형 정보 활용 동의
  - 등록 후 상품 탐색 화면으로 이동
- P1 요구사항
  - 카테고리별 선호 사이즈 세분화
  - 체형 프로필 완성도 표시
- 사용 엔티티
  - `BodyProfile`, `FitPreference`
- 예외 처리
  - 필수값 누락 시 저장 차단
  - 동의 미체크 시 체형 기반 기능 비활성 안내

## 2.3 상품 목록 (`/products`)

- 목표: 상품을 탐색하고 체형 적합도가 높은 상품을 빠르게 찾는다.
- P0 요구사항
  - 검색어 입력
  - 카테고리/브랜드/가격 필터
  - 체형 적합도 높은 순 정렬
  - 인기순, 최신순, 가격순 정렬
  - 상품 카드에 대표 이미지, 브랜드, 상품명, 가격, 체형 적합도 배지 표시
- P1 요구사항
  - 내 사이즈 재고 있음 필터
  - 유사 체형 리뷰 많음 필터
- 사용 엔티티
  - `Product`, `ProductVariant`, `FitScore`
- 예외 처리
  - 검색 결과 없음 Empty State 제공
  - 체형 프로필 미등록 시 적합도 대신 프로필 등록 CTA 표시

## 2.4 상품 상세 (`/products/:productId`)

- 목표: 사용자가 구매 전 사이즈/핏 불확실성을 줄이고 옵션을 선택할 수 있게 한다.
- P0 요구사항
  - 상품 이미지, 브랜드, 상품명, 가격, 소재, 사이즈표 표시
  - 색상/사이즈 옵션 선택
  - 체형 적합도 점수 표시
  - 체형 기반 사이즈/핏 주의사항 표시
  - 유사 체형 리뷰 요약 표시
  - 리뷰 필터 진입
  - 매장 재고 확인 섹션
  - 장바구니 담기
- P1 요구사항
  - 체형별 만족도 분포 차트
  - 대체 추천 상품
  - 요약 근거 리뷰 하이라이트
- 사용 엔티티
  - `Product`, `ProductVariant`, `FitReview`, `ReviewSummary`, `FitScore`, `StoreInventory`
- 예외 처리
  - 유사 체형 리뷰가 없으면 전체 리뷰 요약으로 fallback
  - 선택 옵션 품절 시 장바구니 버튼 비활성화
  - 체형 프로필 미등록 시 일반 리뷰 요약 표시

## 2.5 리뷰 탐색 (`/products/:productId/reviews`)

- 목표: 사용자가 본인과 비슷한 체형의 실제 착용 후기를 직접 확인한다.
- P0 요구사항
  - 전체 리뷰 목록
  - 키 범위, 착용 사이즈, 골격 타입, 선호 핏 필터
  - 핏 평가 필터: 작아요, 잘 맞아요, 커요
  - 리뷰 카드에 별점, 체형 정보, 착용 사이즈, 핏 평가, 텍스트 표시
- P1 요구사항
  - 리뷰 키워드 칩 필터
  - 이미지 리뷰 필터
- 사용 엔티티
  - `FitReview`, `BodyProfile`
- 예외 처리
  - 필터 결과 없음 Empty State 제공

## 2.6 매장 재고 (`/products/:productId/stores`)

- 목표: 온라인에서 본 상품을 가까운 매장에서 입어보거나 구매할 수 있게 한다.
- P0 요구사항
  - 선택한 색상/사이즈 기준 매장별 재고 목록
  - 매장명, 거리, 영업 상태, 보유 수량, 층/존/랙 위치 표시
  - 재고 없음 시 온라인 배송 옵션 안내
- P1 요구사항
  - 지도 UI
  - 매장 방문 예약
- 사용 엔티티
  - `Store`, `StoreInventory`, `ProductVariant`
- 예외 처리
  - 위치 권한이 없으면 기본 지역 기준 매장 표시
  - 재고 조회 실패 시 재시도 UI 제공

## 2.7 장바구니 (`/cart`)

- 목표: 선택한 상품과 옵션을 확인하고 주문 생성으로 이동한다.
- P0 요구사항
  - 상품/옵션/수량 표시
  - 수량 변경/삭제
  - 총 상품 금액 표시
  - 보유 쿠폰 중 적용 가능한 쿠폰 목록 표시
  - 최대 할인 쿠폰 조합 자동 적용
  - 상품별 할인 금액과 주문 전체 할인 금액 표시
  - 쿠폰 적용 제외 사유 표시
  - 주문하기 버튼
- P1 요구사항
  - 사이즈 주의사항 재노출
  - 품절 옵션 자동 안내
- 사용 엔티티
  - `Cart`, `Product`, `ProductVariant`, `Coupon`, `CouponApplication`, `CartPricing`
- 예외 처리
  - 품절 옵션은 주문 진행 차단
  - 적용 가능한 쿠폰이 없으면 할인 없이 총액 표시

## 2.8 주문 생성 (`/checkout`)

- 목표: 상품 선택 이후 주문 생성 흐름을 완성한다.
- P0 요구사항
  - 배송지 입력
  - 주문 상품 확인
  - 자동 적용된 쿠폰과 할인 금액 확인
  - 주문 생성
  - 주문 완료 화면 이동
- P1 요구사항
  - 실제 결제 연동
  - 적립금 적용
- 사용 엔티티
  - `Order`, `OrderLine`, `Cart`
- 예외 처리
  - 배송지 필수값 누락 시 주문 생성 차단

## 3. 셀러 웹

## 3.1 셀러 대시보드 (`/seller`)

- 목표: 입점 상품의 리뷰/반품 상태와 주요 문제를 한눈에 확인한다.
- P0 요구사항
  - 전체 상품 수, 평균 평점, 핏 불만 비율, 반품률 지표 카드
  - 문제 상품 Top 5
  - 최근 인사이트 카드
- P1 요구사항
  - 기간 필터
  - 카테고리별 비교
- 사용 엔티티
  - `SellerInsight`, `FitKeywordStat`, `ReturnReason`

## 3.2 상품 분석 (`/seller/products/:productId`)

- 목표: 특정 상품의 체형별 만족도와 문제 지점을 확인한다.
- P0 요구사항
  - 상품 기본 정보
  - 체형별 평균 평점
  - 사이즈별 핏 평가 분포
  - 긍정/부정 키워드
  - 대표 리뷰
  - 개선 인사이트 카드
- P1 요구사항
  - 상세페이지 수정 제안 문구 생성
  - 경쟁 상품 비교
- 사용 엔티티
  - `Product`, `FitReview`, `FitKeywordStat`, `SellerInsight`

## 3.3 리뷰 분석 (`/seller/reviews`)

- 목표: 리뷰를 체형, 사이즈, 키워드 기준으로 탐색한다.
- P0 요구사항
  - 상품/체형/사이즈/평점/키워드 필터
  - 리뷰 목록
  - 긍정/부정 키워드 요약
- P1 요구사항
  - 리뷰 CSV 다운로드
  - 답변 필요 리뷰 표시
- 사용 엔티티
  - `FitReview`, `FitKeywordStat`

## 3.4 반품 분석 (`/seller/returns`)

- 목표: 반품 원인을 상품/옵션/체형 그룹별로 파악한다.
- P0 요구사항
  - 반품 사유별 비율
  - 상품별 반품률
  - 체형 그룹별 반복 이슈
- P1 요구사항
  - 반품 사유 자동 분류
  - 기간별 추이 차트
- 사용 엔티티
  - `ReturnReason`, `SellerInsight`

## 4. 공통 UI/기술 요구사항

- 고객/셀러 권한 분리
- 반응형 기준: Mobile / Tablet / Desktop
- 상품 이미지 lazy loading
- API 로딩/에러/빈 상태 제공
- 체형 정보와 리뷰어 정보는 비식별 노출
- 주요 CTA 클릭 이벤트 트래킹

## 5. 우선순위 재정렬 요약

- 고객 웹 P0: 체형 프로필, 상품 목록/상세, 리뷰 필터/요약, 매장 재고, 장바구니/쿠폰 자동 적용/주문 생성
- 셀러 웹 P0: 대시보드, 상품 분석, 리뷰 분석, 반품 분석
- P1: 리뷰 작성, 실제 결제, 실제 재고 연동, 자동 인사이트 생성, 리포트 다운로드
