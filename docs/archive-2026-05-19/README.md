# pin-stitch Docs Guide

`docs/` 폴더의 문서를 목적별로 빠르게 찾기 위한 안내 문서입니다.

## 현재 제품 방향

`pin-stitch`는 **체형별 리뷰 분석 기반 패션 커머스 + 셀러 인사이트 플랫폼**입니다.

사용자는 나와 비슷한 체형의 리뷰와 체형 적합도 점수로 구매 실패를 줄이고, 셀러는 리뷰/반품 데이터를 분석해 상품 상세페이지와 옵션 구성을 개선합니다.

## 문서 구성

- `plan.md`
  - 프로젝트 기획서
  - 비전, 문제 정의, MVP 범위, KPI, 로드맵
  - 현재 기준 버전: `v2.0`

- `prd-v2.md`
  - 제품 요구사항 문서
  - 체형 프로필, 리뷰 요약, 체형 적합도, 매장 재고, 셀러 인사이트 요구사항
  - 상태: Draft

- `screen-requirements.md`
  - 고객 웹/셀러 웹 화면별 요구사항 명세
  - P0/P1 우선순위 포함

- `mvp-implementation-plan.md`
  - MVP 구현 계획서
  - 개발 순서, 화면 우선순위, Mock 데이터, 계산 로직, 장바구니/쿠폰/주문 정책

- `project-spec.md`
  - 프로젝트 명세서
  - 기술 스택, 앱 구조, 패키지 구조, 상태 관리, API 구현 기준, 컴포넌트 기준

- `tech-stack.md`
  - 기술 스택 문서
  - 사용할 라이브러리, 상태 관리 기준, API 전략, MVP 이후 도입 후보

- `design-system.md`
  - 디자인 시스템 명세서
  - 색상, 타이포그래피, 레이아웃, 컴포넌트, 고객/셀러 화면 패턴

- `v0-prompts.md`
  - v0 프롬프트 모음
  - 공통 디자인 기준, 공통 컴포넌트, 고객/셀러 화면 생성 프롬프트

- `api-draft.md`
  - API 초안 및 도메인 타입 스펙
  - `BodyProfile`, `FitReview`, `FitScore`, `ReviewSummary`, `StoreInventory`, `SellerInsight` 중심

- `backlog-p1.md`
  - MVP 이후 P1/P2 백로그
  - 리뷰 작성, 반품 사유 입력, 실제 결제, 실제 재고 연동, AI 착장 등

- `prd-template.md`
  - 신규 기능/프로젝트 PRD 작성을 위한 참고 템플릿
  - 현재 MVP 구현에는 직접 사용하지 않음

## 권장 읽기 순서

1. `plan.md`  
   프로젝트 방향과 MVP 범위를 먼저 확인한다.

2. `prd-v2.md`  
   실제 구현해야 할 기능 요구사항과 성공 지표를 확인한다.

3. `screen-requirements.md`  
   고객 웹과 셀러 웹의 화면 단위 요구사항을 확인한다.

4. `mvp-implementation-plan.md`  
   실제 개발 순서와 MVP 완성 기준을 확인한다.

5. `project-spec.md`  
   실제 코드 작성 기준과 기술 스택을 확인한다.

6. `tech-stack.md`  
   사용할 기술과 각 기술의 역할을 확인한다.

7. `design-system.md`  
   화면 스타일과 컴포넌트 구현 기준을 확인한다.

8. `v0-prompts.md`  
   v0로 UI 초안을 만들 때 사용할 프롬프트를 확인한다.

9. `api-draft.md`  
   프론트엔드/백엔드 계약과 도메인 타입을 확인한다.

10. `backlog-p1.md`  
   MVP 이후로 미룬 기능과 다음 스프린트 후보를 확인한다.

## MVP P0 범위

### 고객 웹

- 체형 프로필 등록/수정
- 상품 목록/검색/필터/정렬
- 상품 상세
- 체형별 리뷰 필터링
- 체형 기반 리뷰 요약
- 체형 적합도 점수
- 사이즈/핏 주의사항
- 샘플 매장 재고 안내
- 장바구니/주문 생성
- 장바구니 쿠폰 자동 최적 적용

### 셀러 웹

- 셀러 대시보드
- 상품별 체형 만족도 분석
- 리뷰 키워드 분석
- 반품 사유 샘플 분석
- 상품 개선 인사이트 카드

## 주요 KPI

- 체형 프로필 등록률
- 체형별 리뷰 요약 클릭률
- 상품 상세에서 장바구니 전환율
- 쿠폰 자동 적용 성공률
- 매장 재고 확인 클릭률
- 셀러 대시보드 재방문율
- 핏/사이즈 관련 반품률 변화

## 문서 간 정합성 규칙

- MVP 범위와 KPI는 `prd-v2.md`를 기준으로 관리한다.
- 프로젝트 방향과 로드맵 변경은 `plan.md`에 먼저 반영한다.
- 화면 요구사항 변경 시 `screen-requirements.md`와 `api-draft.md`를 함께 검토한다.
- MVP에서 제외된 기능은 `backlog-p1.md`에 추가한다.
- AI Virtual Try-on, 사진 기반 체형 분석, 실제 재고 연동은 MVP가 아니라 P1/P2로 관리한다.

## 변경 관리 체크리스트

- [ ] 변경한 기능이 P0/P1/P2 중 어디에 속하는지 명확한가
- [ ] 화면 요구사항과 API 스펙이 충돌하지 않는가
- [ ] 체형 정보/리뷰어 정보의 개인정보 노출 위험이 없는가
- [ ] 셀러 인사이트가 실제 실행 가능한 개선 제안으로 이어지는가
- [ ] 고객 구매 판단과 셀러 상품 개선에 직접 기여하는가
